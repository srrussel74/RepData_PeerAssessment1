Imputing missing values

The amount of missing values on data is:

sum(is.na(data))
## [1] 2304

So the percentage of missing values from data is

percentage<-mean(is.na(data))*100

It is 4.3716%.

The strategy to replace NAs with new values is explain as following: Get dataset NAdata as subset with rows contained NA from data:

NAdata<-data[which(is.na(data)),]

To explore how NAs is spreading, we take a look at NAdata. We see there are 8 days where values are missing:

unique(NAdata$date)
## [1] 2012-10-01 2012-10-08 2012-11-01 2012-11-04 2012-11-09 2012-11-10
## [7] 2012-11-14 2012-11-30
## 61 Levels: 2012-10-01 2012-10-02 2012-10-03 2012-10-04 ... 2012-11-30

Given is that total amount of missing values (2304) divided by 8 (days) results to 288, which is total number of interval of a day. So the expectation is that only these whole days are missing values. To confirm this for each of this 8 days, we check it for i=1, which means 2012-10-01:

sum(data$date==NAdata$date[i])
## [1] 288

Repeat the confirmation for each day from NAdata$date (i=2,..,8) shows all same result given as 288.

So in fact we have to insert a new value for all 288 intervals of given day.

The explaination of strategy continues about creating new values: Consider each day i the average number of steps as m[i]:

m<-aggregate(data$step,list(data$date),mean)[,2]

There are NAs in m. If a row of first day from m has got a missing value, then insert the value into m[1] with:

m[1]<-m[2]

The underlying idea for these inserted value is that the missing value is replaced by average value of steps from one neighbouring next day.

For the rest of days, the dataset m is fullfilled, according to:

When there is a missing value at day i, then the inserting value will be an average of mean values from neighbouring, later and earlier, days. If there is a missing value in a day later then calculate mean will be ingored and the value from earlier day will be inserting. That is summarized as:

for(i in 2:length(m))
  {
  if(is.na(m[i]))
    {
    if(is.na(m[i+1]))
      {
      m[i]<-m[i-1]
    }else
      {
        m[i]<-0.5*(m[i+1]+m[i-1])
    }
  }
}

The missing values from a interval in data are replaced by m[i], with i, correspondening date. A set M is formed as set of days where the vaules are missing:

M<-unique(NAdata$date)

Create new dataset Ndata where NAs are replaced as following:

Ndata<-data
for( day in M ){
  Ndata$steps[which(Ndata$date==day)]<-
    m[which(unique(Ndata$date)==as.character(day))]
}

The procedure after creating Ndata (after first line of code) is: it is given that the length of m is equal to 61 days. So does unique(Ndata$date). A value from m is corresponding to a day. So for set of Ndata$set corresponding to same day is filled by the value from m. Remind that M contains days where values are missing. So procedure focuses on these days.

And median of total number of steps per day is:

These values differs from mean and median of number of steps from data where NAs are omitting. And new dataset has got more values, so average value decreases. Despite that the sum of the number of steps increases. That is because of the fact that the increasing sum does relatively not have an effect above the effect from the changing length of dataset. This combined by fact that the new values for sum are based on averages. Check earlier chunk code for creating dataset m. About median, there also are more values added to Ndata. Conclusion is that the adding new values based on the above described strategy brings more effect from lower values on estimates of total daily number of steps.

Are there differences in activity patterns between weekdays and weekends?

Using dataset Ndata, a new dataset W ,contained factor variables with levels Weekday and Weekend, is formed by:

W<-weekdays(as.Date(Ndata$date))

W[which((W=="Saturday")|(W=="Sunday"))]<-"weekend"
W[which((W!="weekend"))]<-"weekday"

W<-factor(W, level=c("weekend","weekday"))

The level indicates a date, as being a day from week or weekend.

Wsi<-aggregate(Ndata$steps,list(interval=Ndata$interval, W=W),mean)
colnames(Wsi)<-c("interval","W","steps")

The dataset Wsi is average number of steps per interval divided into two groups by two kinds of days: weekday and weekend.

The plots of the two groups from dataset Wsi are presented by:
